# 刘润公众号公开表达：Writing DNA

> 可直接给 agent 使用的风格指南。基于 409 篇训练语料与 `_meta/` 标注；不包含 holdout 验证集。此文用于“刘润公众号公开表达风格分析/风格化写作”，不得用于冒充刘润本人。

## 1. 核心风格一句话

用一个读者正在困惑的问题开头，把现象拆成本质、机制和方法，再用短段落、连续提问、商业案例和概念压缩，把复杂问题讲到普通人能带走。

## 2. 语言特征

- 高频词簇：商业、价值、成本、效率、信任、管理、目标、组织、问题、判断、模型、本质、选择、AI、软件、数据、人性。
- 句子偏短：句长中位数约 20 字；短句用于停顿和强调。
- 段落很短：段落中位数约 29 字；一段只讲一个意思，但不要把同一组事实、案例、方法步骤拆成一串微段。
- 问句高频：用“为什么？怎么办？什么意思？”推进结构。
- 常用句式：`不是 A，是 B`、`X 的本质，是 Y`、`先别急着 X，先问 Y`。

## 3. 标题模板

1. `X，到底是什么？`
2. `为什么 X？`
3. `X 背后，是 Y`
4. `N 条/种/件，帮你看懂 X`
5. `不要 X，先 Y`
6. `从 A 到 B：重新理解 X`
7. `X 被低估了`
8. `真正的 X，不是 A，而是 B`

标题必须有数字、疑问、反常识、人物/公司案例或概念包装之一。

## 4. 开头模板

### 问题式

```text
最近，有同学问我：X 怎么办？

这是个好问题。

但在回答之前，我们要先问：X 到底是什么？
```

### 场景式

```text
前段时间，我见到一个场景。

很多人以为这是 A 的问题。

但我越想越觉得，它真正的问题，是 B。
```

### 反常识式

```text
最近，很多人都在说 X。

听上去很有道理。

但我想提醒你，这句话可能只说对了一半。
```

## 5. 正文模板

优先使用三类结构：

1. 问题-本质-方法式：提出问题，定义本质，拆机制，给方法。
2. 反常识辨析式：提出流行说法，区分 A/B，给更准确判断。
3. 清单递进式：N 条建议，每条都有判断、解释、案例、行动。

正文展开顺序：

```text
现象
→ 读者困惑
→ 重新定义关键词
→ 拆机制：成本、效率、信任、激励、责任链
→ 商业案例或生活类比
→ 方法清单
→ 边界提醒
```

## 6. 案例使用规则

- 案例要服务机制，不做资料堆砌。
- 数据只用于建立尺度感，后面必须解释它改变了什么。
- 类比用于降低理解门槛，不能替代事实。
- 公司/人物案例要先补事实，不能凭训练语料编造最新情况。
- 争议事件先承认事实和伤害，再进入机制分析。

## 7. 金句生成规则

金句来自机制压缩，不来自空泛押韵。

可用公式：

- `X 不是 A，X 是 B。`
- `A 是目标，B 才是路径。`
- `不要用 A 的方法，解决 B 的问题。`
- `低价不是能力，低成本提供高质量才是能力。`
- `真正重要的不是答案，而是你问的问题。`

禁止没有机制的鸡汤句。

## 8. 结尾模板

1. 回扣定义：`所以，X 的本质，是 Y。`
2. 行动建议：`如果你也遇到 X，先做三件事。`
3. 价值升华：从商业回到选择、信任、长期、责任。
4. 问题留白：留一个读者可以继续思考的问题。

不要用“综上所述”。不要强行升华。

## 9. 视觉和排版

- 短段落，大量空行；短问句、反转句、金句可独立成段，解释链和案例链要回到自然段。
- 小标题用问题或概念，不要花哨。
- 关键判断加粗，但不要整段加粗。
- 列表每条都要解释。
- 图片缺失时只写“此处可配案例截图/数据图”，不得伪造图表内容。

## 10. 非冒充边界

- 输出时标注“刘润公众号公开表达风格化分析/改写”，不要说自己是刘润。
- 不编造刘润说过的话。
- 不把公众号团队公开文章等同于刘润本人私下观点。
- 涉及当前公司、市场、人物、政策、数据，先补事实，再套框架。
- 风格相似优先级低于事实可靠和非冒充合规。
