# 文章结构模板

## A. 多店合集

```text
总钩子：这次为什么整理这批店
→ 选择标准：社区老店/县城味/某片区/某单品
→ 店1：一句定位 + 环境/来历 + 必点 + 口感判断 + Tips
→ 店2/3/4 同结构
→ 总结：适合谁去、怎么安排、评论补充
```

## B. 单店深挖

```text
反差入口：藏、排队、熟人带、开很多年、某个菜强
→ 到店现场：位置、环境、人群、老板或店规
→ 菜品主线：招牌菜先写，再写搭配/隐藏点法
→ 判断：值得去的理由 + 不适合谁/注意事项
→ Tips：店名、时间、地址、点单建议
```

## C. 街区地图/城市记忆

```text
片区情绪：这里为什么值得重新看
→ 路线感：从地铁/学校/厂区/老小区切入
→ 3-8 个点位：每个点位给一个明确吃点
→ 城市关系：谁在这里吃、这里和成都记忆的关系
→ 行动入口：适合半日/晚饭/宵夜/带朋友
```

## D. 测评指南

```text
测评问题：读者到底想知道什么
→ 规则：买了什么、怎么比、看哪些维度
→ 逐项结果：口味/价格/便利/复购意愿
→ 排名或推荐：直接给选择建议
→ 例外：不推荐或只适合特定人群
```
