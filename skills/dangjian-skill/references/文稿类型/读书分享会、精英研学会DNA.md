# 读书分享会、精英研学会 DNA

## 适用场景

读书分享会、精英研学会、员工学习交流、主题分享、岗位经验交流、学习型党组织建设。

## 稳定结构

1. 活动期数 + 时间 + 主体。
2. 活动目的：搭建交流平台、促进互学互鉴、提升履职能力。
3. 分享人和分享主题：按人名/部门/书目/主题展开。
4. 岗位转化：把读书心得转成内容创作、服务沟通、执行力、运营能力。
5. 末段：浓厚学习氛围 + 常态化开展 + 以学促干。

## 写法要点

- 读书不是为读书而读书，必须回到岗位实践。
- 可列多个分享人，但每个人只抓一个核心观点。
- 语气比领导宣传轻一点，但仍保持组织简报口径。
- APP 发帖场景可稍有主持人口吻；正式简报不要太网感。
- 多期读书分享会不要拉成长篇总结。已有主题、书目、分享人时，控制在 500-700 字，按“活动目的 -> 2-3 个代表分享 -> 岗位转化 -> 下一步”收束。
- 书记表态、学习型党组织、以学促干只能各出现一次；没有素材支撑时不要额外拔高。
- 用户只说“写简报”时不要自动加“练团子党支部/简报/第X期”抬头；只有用户明确要正式简报头或提供期数时才加。

## 标题公式

- `练团子党支部组织第X期读书分享会`
- `第X期精英研学会 + 学习/成长/赋能主题`
- `以学促干 + 凝心聚力/实干担当`

## 反例

- 只写“大家积极分享”，不写谁分享了什么。
- 只复述书名，不写岗位联系。
- 结尾泛泛“收获满满”，没有学习型组织承接。
- 每位分享人都写成同样的“结合岗位实践、提供参考”，导致段落像模板复制。
- 为了像正式文件而强行加简报头，让正文显得套壳。
