# 语料质量报告

## 目标与口径

- 对象：大宜宾 APP 多小编写稿 Skill。
- 目标小编：流浪啊, 大宜宾雯雯, 练团长, 泡泡呀, 采采呀, 双双呀, 大宜宾-梦竹。
- 语料目录：`/Users/lixin/Documents/大宜宾/大宜宾知识库整理/大宜宾 app 资料库`。
- 取样口径：每个小编独立排序，不做总池混排；先排除运营活动帖，再取该小编自己的互动数据前 60%。
- 互动分：`浏览 + 回复*100 + 点评*20 + 分享*10 + total_heat`。回复权重更高，因为 APP 帖核心价值是评论区启动。
- 活动帖排除：评论送票、抢票、抽奖、红包福利、游园会、参与领取、抢楼等运营型活动；不排除新闻里的“活动轨迹”、政务“意见征集”、普通招聘报名。
- holdout：每个小编从自己的 top60 池冻结 3 篇，共 21 篇；训练集共 1506 篇。

## 结论

1. 该语料足够支持账号基线、文稿类型 DNA 和 7 个小编 DNA。每个目标小编训练样本均超过 20 篇。
2. 大宜宾 APP 的核心不是单一文风，而是“本地议题 + 评论区启动 + 小编出手方式”的路由系统。
3. `大宜宾雯雯` 与 `大宜宾-梦竹` 的路线更偏政务/招聘信息分发，不能强行写成流浪啊/泡泡呀那类互动口吻。
4. `泡泡呀` 的样本里运营活动较多，已过滤明显福利/送票/抢票帖；保留民生讨论和服务型稿。
