# 写稿流程操作手册

## 1. 输入素材盘点

先列三栏：

- 已确认：时间、产品、功能、价格、入口、截图、结果、来源。
- 可推断：可能影响、适合人群、短期趋势。
- 禁止补：亲测经历、朋友对话、购买行为、内部消息、未给出的数据。

## 2. 选主角

一篇文章只能有一个主角：一个工具、一个模型、一个功能、一个乱象、一个个人经历。其他东西只能做配角。

## 3. 写开头

四步内完成：

```text
具体时间/场景
-> 我看到/试到/被问到什么
-> 第一情绪
-> 这篇要帮你解决什么
```

## 4. 搭正文

每个小节按这个顺序写：

```text
小节判断
-> 证据/截图/案例/步骤
-> 人话解释
-> 我的判断
-> 小坑或边界
```

## 5. 写结尾

不要机械总结所有功能。结尾回答一个更大的问题：这东西让谁更强、让谁更难、让什么信任关系变脆、普通人接下来该做什么。

## 6. 出稿自检

- 标题是否有对象和张力？
- 前 5 段是否出现了具体场景，而不是抽象背景？
- 每个强判断后面是否有证据？
- 技术词是否被翻译成人话？
- 是否有至少一个“普通人怎么用/怎么避坑”的落点？
- 是否没有冒充作者本人或伪造亲测？
