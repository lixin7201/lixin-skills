# 标题 DNA

标题目标：让读者一眼知道对象，同时被一个强判断绊住。

## 公式库

| 公式 | 用法 |
|---|---|
| X 的敌人只有 Y | 公司、品牌自我误判 |
| X，Y 的神 | 消费品/工具被赋予荒诞用途 |
| X 是怕我 Y 吗？ | 个人体验型产品吐槽 |
| X 凭什么这么 Y？ | 公司动作、定价、产品发布 |
| X 怎么成了 Y？ | 反常识流行现象 |
| X 不是 A，是 B | 影视、商业、社会议题反转 |
| X 给我看傻眼了 | 事件或产品强情绪入口 |
| X 承受了太多 | 把普通产品写成荒诞英雄 |
| 多少人把 X 当 Y？ | 用户误用、场景迁移 |
| X 要毁了 Y 吗？ | 竞争格局、品类变化 |
| X 的本质，是 Y | 深拆机制文 |
| X 面前，Y 都是 Z | 对比型商业判断 |
| X 哪里来的胆子？ | 定价、扩张、战略动作 |
| X 笑了 / X 慌了 / X 傻眼了 | 竞争双方戏剧化 |
| X 救了 Y 的命 | 工具、消费品、平台场景 |

## 标题检查

- 有明确对象。
- 有判断，不只是信息。
- 读者能感觉“这话有点过，但我想看他怎么圆回来”。
- 不用未知事实当标题。
- 不连续使用同一种公式，避免模板味。

## 禁区

- 不用“深度解析”“万字长文”“一文看懂”这类报告标题。
- 不用空泛情绪词做标题主体。
- 不用原作者已有标题的改字版本。
