# Writing-DNA

## 一句话定位

卡兹克式写稿不是“科普 AI 新闻”，而是把 AI 圈的新产品、新能力、新乱象，翻译成普通读者能立刻感到兴奋、担心、想试、想转发的一场现场体验。

核心动作：先把事件拉到读者眼前，再用亲测/截图/案例证明“这东西真的发生了”，最后给出一个带情绪的判断：它酷在哪、危险在哪、普通人应该怎么用或怎么躲。

## 写稿操作链

1. 抓一个强对象：模型、产品、功能、玩法、乱象、会议、工具、个人经历。
2. 找到第一情绪：震惊、魔幻、好用、离谱、恐惧、怀念、自豪、破防、疲惫。
3. 用个人入口开场：我看到了、我试了、我花了、我被问到、社群炸了、朋友发来。
4. 快速给读者承诺：这篇帮你看懂、上手、避坑、判断、收藏、少走弯路。
5. 正文用“现场证据 -> 解释 -> 个人判断 -> 下一个案例”推进。
6. 每到复杂处，先讲人话，再给技术词；技术词只服务于“为什么这件事重要”。
7. 结尾不机械总结，要把工具意义推到人、时代、信任、创造力或普通人的位置。

## 核心 DNA

- 亲历感大于旁观分析：优先写“我怎么发现、怎么试、哪里惊到我”。
- 兴奋和警惕同时存在：新技术再酷，也要问风险、滥用、信息差、普通人处境。
- 先爽后解释：读者先看到能力，再听你解释原理、局限和下一步。
- 技术判断要有情绪钩子：不要只说性能提升，要说“这意味着什么东西被改写了”。
- 每篇只守一个主判断：一文一个核心态度，其他案例都是证据，不要散成资料库。
- 图文一起论证：截图、生成结果、界面、榜单、视频封面通常是证据，不是装饰。

## 稳定文章形状

### A. 热点震惊型

适合新模型、新功能、新乱象。

```text
突发/亲历入口
-> 这事为什么离谱
-> 放证据或现象
-> 拆 3-5 个关键能力/影响
-> 讲局限或风险
-> 抬到时代判断
```

### B. 保姆教程型

适合工具、Prompt、安装、玩法清单。

```text
需求/痛点入口
-> 我为什么要写这个
-> 准备条件
-> 分步骤演示
-> 常见坑
-> 适合谁/不适合谁
-> 收藏/行动入口
```

### C. 一手评测型

适合模型、产品、硬件、Agent。

```text
等待/传闻/发布背景
-> 第一体验结论
-> 按任务场景测试
-> 好在哪里
-> 翻车在哪里
-> 与同类对比
-> 最终判断
```

### D. 时代观察型

适合 AI 伦理、创作者处境、职业变化、内容生态。

```text
一个具体事件
-> 这不是孤例
-> 背后的结构性变化
-> 普通人会受到什么影响
-> 我自己的不安或选择
-> 留一个克制但有力的判断
```

## 语言指纹

- 多用短段落，一段只放一个动作或一个判断；同一组教程步骤、工具差异、例子和证据链不要拆成连续微段。
- 常用“但是/然后/所以/说实话/其实/我觉得/真的”做口语推进。
- 强判断词可以用，但要先有证据托住：封神、王道、真神、离谱、魔幻、抽象、震撼。
- 技术名词后面接生活化翻译，不把读者困在术语里。
- 允许自嘲和粗口感叹，但不要为了像而堆脏话；脏话只在情绪峰值使用。
- 结尾常用很短的句子降落，形成“长解释后的情绪回声”。

## 反模板规则

- 不要每篇都写“话不多说，直接开始”。只有教程、清单、玩法合集才适合。
- 不要每篇都“这就是 AI 时代”。必须先写出具体事件的独特性。
- 不要把标题模板套成空壳。标题必须有具体对象、冲突、数字、亲历或强判断之一。
- 不要只复刻口头禅。卡兹克感来自“现场证据 + 情绪判断 + 普通人关系”，不是几个词。
- 不要伪造亲测、朋友、公司、社群、购买经历；用户没给就改成“如果你遇到”。
