# 刘润公众号公开表达：写作视角与认知框架

> 方法：按 `writing-dna-skill` L3-L5 提取。统计对象为训练语料；未使用 holdout 验证集。

## L3 选题逻辑

### 选题优先级

1. 商业世界中正在变化的结构：AI、软件、零售、电商、平台、低价、组织形态。
2. 企业管理中的真实痛点：目标、分钱、中层、老板、员工离职、办公室政治、责任链。
3. 热点事件背后的机制：谣言、AI误判、公司风波、政策变化、行业内卷。
4. 普通人的成长问题：学习、选择、深度思考、时间、心力、职业路径。
5. 可被重新定义的日常概念：战略、商业模式、信任、生产率、财务报表、名片、超市。

### 切入时机

- 热点刚发生时，不直接追情绪，追“这件事背后的机制”。
- 年中、年末、节日、考试、618、政策节点，适合做复盘和行动建议。
- 新技术出现时，不只讲工具，而讲它改变什么角色、流程、价值链。
- 公司案例被广泛讨论时，用它解释一个更一般的商业规律。

### 不优先写什么

- 只有情绪没有机制的站队文。
- 只有资料堆砌没有判断的资讯文。
- 纯私人八卦和无法验证的内部消息。
- 只为追热点而不能沉淀框架的话题。

## L4 素材策略

训练统计显示，素材类型高频为商业案例 291 篇、数据/报告 260 篇、一手观察/咨询场景 188 篇、类比 177 篇。

### 商业案例使用方式

- 先用案例打开兴趣，再抽象出机制。
- 案例常来自公司、品牌、平台、管理现场、行业事件。
- 案例不是为了证明“我知道很多”，而是为了让抽象概念可感知。
- 同一个机制常用多个案例互相校验，避免单点结论。

### 数据使用方式

- 数据用于建立尺度感，如金额、人数、增速、成本、损失。
- 不把数据做成学术证明，而是用来回答“这事到底有多大”。
- 数据后面必须接解释：数字说明哪条链路变了。

### 类比策略

常用类比方向：

- 商业问题类比成生活选择。
- 组织系统类比成机器、钟、链条、坐标系。
- 技术变化类比成电力、软件、工具、耗材。
- 信任和交易类比成邻里关系、多次博弈。

类比规则：类比只负责降低理解门槛，不能替代事实。

## L5 认知框架

### 核心命题

1. 商业的底层不是热闹概念，而是价值交换和交易成本。
2. 管理的底层不是管人，而是让目标、责任、信息、激励对齐。
3. 技术的价值不是炫技，而是重做流程、角色和价值链。
4. 好表达不是观点堆砌，而是事实、结构和读者关系。
5. 成长不是复制成功者，而是在自己的原始资本上选择打法。
6. 低信任环境会让所有东西变贵，长期信任本身就是资产。

### 金句生成方式

刘润式金句通常不是押韵，而是“概念压缩”：

1. 定义压缩：`X 不是 A，X 是 B。`
2. 关系压缩：`A 是目标，B 才是路径。`
3. 机制压缩：`所谓 X，就是把外部成本变成内部成本。`
4. 选择压缩：`不是要不要做，而是你用什么打法做。`
5. 边界压缩：`这句话对，但只在某个前提下对。`

生成金句前，必须先有机制。没有机制的句子，即使像金句，也是空话。

## 写作视角

- 从读者困惑进入，而不是从作者知识进入。
- 从现象进入本质，从情绪进入机制，从案例进入框架。
- 对公司和人物不轻易道德审判，先看约束和激励。
- 对普通人不灌鸡汤，先问资源、路径和可控变量。
- 对新技术不盲目乐观，也不恐慌，先问价值链重组。

## 改写规则

把普通文章改成刘润公众号公开表达风格时：

1. 先找一个读者正在问的问题。
2. 把原观点改成“问题-本质-机制-方法”。
3. 每 3-5 段插入一个短问句或短判断。
4. 抽象词后补案例或类比。
5. 去掉学术腔、政策腔、咨询 PPT 腔。
6. 保留诚实边界，不把推断写成事实。
7. 明确标注“风格化改写”，不冒充刘润本人。
