# 刘润公众号公开表达：视觉风格指南

> 方法：按 `writing-dna-skill` L6 提取。统计对象为训练语料 Markdown。说明：Markdown 中存在大量 `./assets/...` 图片引用，但本地语料目录未包含对应 `assets/` 文件夹，因此本轮无法直接查看原图内容。以下结论基于图片数量、位置、周边文本和 Markdown 排版标记，图片内容功能需在补齐 assets 后复核。

## 1. 图片密度

- 语料中图片引用很密集，许多长文包含十几到几十张图。
- 文章开头通常有公众号头图、账号标识、分隔图或导流图。
- 正文中图片常出现在小节开头、案例转场、数据/截图证据位置。
- 部分图片显然是固定模板素材，训练时不能把它们当正文论证。

## 2. 图片功能分类（基于引用位置推断）

| 类型 | 作用 | 使用建议 |
|---|---|---|
| 头图/账号图 | 建立公众号身份和栏目感 | 复写文本时不用生成 |
| 分隔图 | 标记章节和视觉喘息 | 可用小标题和空行替代 |
| 案例截图 | 承载事实、聊天、数据或界面证据 | 写作时要用文字补足证据，不假装看过截图 |
| 数据图表 | 提供尺度感和可信度 | 若没有真实图表，改用文字列出数据来源 |
| 人物/公司图 | 强化案例对象识别 | 可在写作说明中标注“此处可配公司/人物图” |
| 商品/生活图 | 适合生活方式或茶酒内容 | 商业分析类少用 |

## 3. 小标题风格

- 小标题短，通常是问题句或概念句。
- 常见形式：`到底什么是 X？`、`为什么？`、`怎么办？`、`最后的话`。
- 小标题承担结构转场，不追求文学化。
- 有时用加粗句替代显式标题。

## 4. 加粗规则

加粗是刘润公众号公开文章的重要节奏工具。

常见加粗对象：

1. 关键问题：`怎么办？`
2. 核心定义：`X 的本质，是 Y。`
3. 反转判断：`不是 A，是 B。`
4. 方法清单里的关键动作。
5. 结尾金句或价值回扣。

使用频率：每个小节 1-3 处即可。不要整段加粗。

## 5. 列表与编号

- 数字列表用于方法论、建议、管理心法。
- 列表项前常有一句总领，不直接进入清单。
- 列表中的每一条都需要解释，不只是短标题。
- 对商业分析，优先用“问题清单”而不是 PPT 式 bullet。

## 6. 排版密度

- 段落短，空行多。
- 一段只承载一个意思。
- 问句可单独成段。
- 重要判断可单独成段。
- 同一组事实、案例、方法和机制解释不要一项一段；它们应合并为一个自然段或正式列表。
- 正文应有口语讲述感，不要写成连续大段论文。

## 7. 视觉复刻边界

- 本轮未能打开原始图片内容，不能声称完成图片语义级复刻。
- 如果要做公众号排版级复刻，需要补齐 assets 或原始 HTML，再复核图片内容、字体层级、色彩和图文协作。
- 在 Skill 中使用视觉规则时，只能要求“短段落、加粗、小标题、图片占位说明”，不得伪造截图或数据图。
