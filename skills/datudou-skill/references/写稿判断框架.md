# 写稿判断框架

## 核心判断

一篇大土豆式文章必须回答四个问题：

1. 这事跟普通家庭的真实处境有什么关系？
2. 这背后是周期变化、门槛变化、人口变化，还是钱和精力的重新分配？
3. 普通人最容易误判哪里？
4. 最后应该更激进，还是更保守、更怂一点？

## 选题成立条件

至少命中两项才值得写：

| 条件 | 可写信号 |
|---|---|
| 现实账本 | 能换算成钱、房、分数、时间、身体、社保、学位、通勤 |
| 北京坐标 | 涉及北京房产、教育、家庭成本、区位差异或中产处境 |
| 代际变化 | 80 后、90 后、年轻人、父母、孩子的选择逻辑不同 |
| 反常识 | 主流热闹处可能是坑，冷门处可能是退路 |
| 亲历入口 | 作者见过、被问过、自己经历过、身边人正在发生 |
| 自保价值 | 能让读者少亏钱、少焦虑、少折腾、少透支 |

## 默认判断顺序

1. 先看趋势：人口、收入、就业、房价、政策、教育供需有没有变。
2. 再看个体：普通家庭有没有纠错能力，试错成本高不高。
3. 再看账本：这个选择会不会消耗现金流、身体、时间和家庭关系。
4. 再看人性：人会跟风，会用现状预测未来，会高估自己抗风险能力。
5. 最后给建议：不是万能建议，而是一种偏保守的行动边界。

## 观点强度

| 强度 | 使用场景 | 表达方式 |
|---|---|---|
| 强判断 | 有明确数据、经历或趋势支撑 | “大概率”“很难”“基本就是” |
| 中判断 | 来自经验归纳 | “我觉得”“在我看来”“比较客观” |
| 弱判断 | 事实不完整或只适合北京 | “未必”“一家之言”“大家可以参考” |

## 一票否决

- 用户只给一个空泛观点，没有事实、场景或对象：先追问，不硬写长文。
- 需要最新政策、学校、楼盘、价格、招生数据但用户未给材料：标注待核实。
- 只想要情绪宣泄、立场攻击或冒充作者本人：改为按写稿方式输出。
