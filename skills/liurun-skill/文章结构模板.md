## 模板 1：问题-本质-方法式

适用：商业分析、组织管理、产品增长、热点事件。

结构：

1. 开头：提出一个读者也在困惑的问题。
2. 第一转折：先说“很多人以为问题是 A，其实不是”。
3. 定义：问“到底什么是 X？”或“X 的本质是什么？”
4. 拆机制：从成本、效率、信任、激励、责任链中找关键变量。
5. 举案例：给 1-3 个商业案例或管理场景。
6. 给方法：把案例归纳为可执行动作。
7. 结尾：回扣本质，提醒边界。

骨架：

```text
最近，很多人问我：X 怎么办？

这是个好问题。

但在回答之前，我们要先问：X 到底是什么？

很多人把 X 当成 A。
不是。
X 的本质，是 B。

为什么？我给你讲一个例子。

例子之后，回到机制：谁的成本变了？谁的收益变了？谁的责任没有被看见？

所以，如果你也遇到 X，先做三件事。
第一……
第二……
第三……

最后，记住一句话：不要试图用 A 解决 B。
```

## 模板 2：反常识辨析式

适用：纠正流行误解、热点观点、AI/技术、低价、内卷、成功学。

结构：

1. 开头：抛出一个流行说法。
2. 反转：明确说“我不太同意”或“这句话只说对了一半”。
3. 拆词：重新定义关键词。
4. 对比：A 看起来对，但忽略了 B。
5. 历史/行业类比：说明这个误解为什么常见。
6. 新判断：给出更准确的表达。
7. 边界：承认在某些情况下原说法仍成立。

骨架：

```text
最近，有一种说法很火：X。

听上去很有道理。
但我想提醒你，这句话可能只说对了一半。

因为它混淆了两个东西：A 和 B。
A 是……
B 是……

如果你只看到 A，就会得出结论 C。
但如果你看到 B，会发现真正重要的是 D。

所以，更准确的说法不是 X，而是 Y。
```

## 模板 3：清单递进式

适用：管理心法、学习方法、职业建议、年度复盘。

结构：

1. 开头：承认读者焦虑或问题普遍存在。
2. 总领：给出“今天分享 N 条”。
3. 每条结构固定：判断句 + 解释 + 案例/类比 + 行动建议。
4. 中段插入一次“更底层”的提醒，避免清单碎片化。
5. 结尾：把 N 条收束到一个核心命题。

骨架：

```text
如果你最近也觉得 X，不妨先停一下。

今天，我想和你分享 N 条思考。

第一，……
这句话听起来简单，但背后是……

第二，……
很多人会忽略……

第三，……
真正困难的不是……而是……

这 N 条背后，其实都是同一件事：Y。
```

## 模板 4：案例复盘式

适用：公司、品牌、平台、政策或产业事件。

结构：

1. 开头：说明“为什么现在值得看这个案例”。
2. 背景：给出关键事实，不堆资料。
3. 拆阶段：过去怎么做，现在遇到什么变化。
4. 拆变量：成本、效率、信任、组织、价值链哪一环改变。
5. 推广：从一个公司抽象出一个行业规律。
6. 结尾：给创业者、管理者或普通人的启发。

## 模板 5：对话/咨询场景式

适用：读者提问、企业家参访、管理困惑、个人成长。

结构：

1. “有同学问我/有读者问我/我见过一个客户”。
2. 把对方的问题复述得更尖锐。
3. 先不回答，换一个问题。
4. 用故事或类比让读者自己意识到问题。
5. 给出框架和具体动作。

## 过渡方式

- 用“但是”承接反转。
- 用“什么意思？”进入解释。
- 用“我给你举个例子”进入案例。
- 用“所以”收束机制。
- 用“最后的话”或“最后”进入价值回扣。

## 结构禁忌

- 不要只写“首先、其次、最后”。
- 不要让每节都同等长度，关键节可以展开，次要节可以短。
- 不要把案例堆成资料卡，案例必须服务一个机制判断。
- 不要用没有事实支撑的金句做主干。
