# 承德 skill

用于把素材写成 `承德新鲜工作室`风格的公众号稿。先路由，再写稿；不冒充小编本人，不复制原文，不编造事实。

## 触发词

- 用我们公众号风格写
- 按这个公众号风格改
- 写成民生稿
- 写成政务稿
- 写成活动稿
- 用小编A的风格写
- 用事儿哥的风格写
- 这篇哪里不像我们公众号
- 这篇哪里不像小编A
- 这篇哪里不像事儿哥
- 按我们公众号常用风格重写
- 标题写得像承德新鲜工作室

## 必读 DNA

每次执行前至少读取：

1. `账号总风格.md`
2. `账号选题判断框架.md`
3. `账号语言底线.md`
4. 对应的 `文稿类型/*DNA.md`
5. 如果用户指定稳定小编：读取 `小编风格/事儿哥-DNA.md`
6. 如果用户指定样本不足小编：读取 `小编风格/小编样本不足.md`
7. 终稿去 AI 味时参考 `/Users/lixin/.openclaw/workspace/skills/de-ai-preserve-voice/SKILL.md`
8. 段落节奏以原文为准：承德长文段落中位约 49 字，短判断可单独成段，事实说明、城市对比、政策解释和现场叙述不要拆成连续微段。

当前已生成文稿类型：民生稿, 政务稿, 活动稿, 商家稿, 人物稿, 情感稿, 招聘稿, 文旅稿, 教育稿, 突发短消息, 服务提醒, 评论观点；另有 `深度长文DNA.md` 作为长稿增强路线，需要叠加在一个主类型上使用。

## 写文章标题先行规则

只要用户要求写文章、完整稿、公众号稿、本地号稿或按承德新鲜工作室风格成稿，先输出标题备选，再写正文；这条规则覆盖后文里只写单个标题的旧默认格式。

写正文前，必须站在承德新鲜工作室标题风格里，从多个角度起标题：本地对象/地名、读者收益、冲突反差、服务提醒、城市情绪或评论观点。给 3-5 个标题备选，标出“推荐标题”，再进入正文。

标题不得暴露生产链路，不写“资料、原文、素材、二创、改写、根据你提供”；高风险事实不得写成确定标题。

## 路由流程

1. 判断用户是否要承德新鲜工作室整体风格。若是，加载账号总风格。
2. 判断用户是否指定文稿类型。指定则读取对应类型 DNA；未指定则根据素材判断，无法判断时只问一个问题确认类型。
3. 判断用户是否指定小编。
4. 检查素材是否足够支撑该路线。
5. 选择 DNA 组合后再写稿、改稿、诊断或优化标题。

## 路由规则

| 用户指定 | 使用 DNA | 处理 |
|---|---|---|
| 未指定类型 | 账号总风格 + 自动判断类型 | 判断不清时先问类型 |
| 指定类型，未指定小编 | 账号总风格 + 文稿类型 DNA | 正常写稿 |
| 指定小编，未指定类型 | 账号总风格 + 小编 DNA + 自动判断类型 | 只支持 `事儿哥` 稳定个人 DNA |
| 同时指定类型和小编 | 账号总风格 + 文稿类型 DNA + 小编 DNA | 小编不足时降级 |
| 小编样本不足 | 账号总风格 + 文稿类型 DNA | 明确提醒不能稳定复刻该小编 |
| 指定深度长文 | 账号总风格 + 主类型 DNA + 深度长文 DNA | 先判断主类型；材料不足时不写长文 |

## 材料缺口检查

写完整公众号稿前必须检查：

- 是否有具体对象：人、地点、事件、时间、商家、政策或活动。
- 是否有至少 3 个不可编造的事实点。
- 是否有来源：官方链接、截图、现场图、采访、投稿、公告或明确说明。
- 是否知道读者收益：避坑、服务、理解、情绪共鸣或行动路径。
- 是否涉及高风险：政策、医疗、法律、维权、未成年人、灾害、商家权益。

材料不足时，不写完整长文。输出“缺口清单 + 可写的短提醒/线索征集/标题方向”。

## 写稿流程

1. 建事实台账：已确认、待核验、禁止脑补。
2. 选路线：账号总风格 + 类型 DNA + 可用小编 DNA。
3. 写标题：前 10 字放承德或核心对象，给冲突/收益/反差。
4. 写开头：三段内交代对象、冲突、读者为什么要看。
5. 写正文：事实推进，不堆材料；每一段都服务读者理解。
6. 写结尾：给判断、动作、提醒或评论问题。
7. 自检：风格像、类型像、事实稳、没有冒充。

## 终稿去 AI 味保真补丁

写稿和改稿最后执行 `de-ai-preserve-voice`，但承德本地账号路线和类型 DNA 优先。

- 先保留账号总风格、文稿类型 DNA、事儿哥线、承德读者关系和本地事实台账。
- 成稿必须去掉二创痕迹：不要写“资料中提到、原文中提到、根据你提供的素材、文中写到”；把素材里的事实转成承德本地账号自己的叙述。
- 只处理官话、空泛拔高、万能公众号口吻、机械反问和无来源权威感。
- 去味不能补本地事实：地点、路线、时间、价格、采访、评论、官方回复、阅读量都必须来自材料。
- 不把民生、政务、商家、文旅、人物稿写成同一种“自然口语稿”。
- 如果普通正文连续 5 段以上都低于 20 字，或像标题清单一样一项一段，优先合并同功能事实链、解释链和城市情绪链。
- 去味后如果类型 DNA 变淡、本地读者关系变弱或事实边界被改动，回滚该处。

## 改稿流程

1. 先判断原稿路线：类型、小编、素材强弱。
2. 标出不像账号的地方：标题、开头、结构、语气、本地锚点、事实边界。
3. 标出不像类型的地方：对照对应 DNA。
4. 若指定 `事儿哥`，标出不像个人 DNA 的地方。
5. 先给具体改法，再输出改后稿。

## 审稿诊断流程

按以下顺序诊断：

1. 账号相似度：是否有承德本地利益、强判断、人味和事实刹车。
2. 类型相似度：是否符合对应 DNA 的结构和素材规则。
3. 小编相似度：只对 `事儿哥`打稳定个人风格分。
4. 事实可靠性：哪些句子缺来源、过度推断或高风险。
5. 可改动作：每个问题给一句可执行修改建议。

输出格式：

```markdown
## 诊断结论
- 账号相似度：x/100
- 类型相似度：x/100
- 小编相似度：x/100 或 样本不足

## 不像在哪里
| 位置 | 问题 | 为什么不像 | 改法 |
|---|---|---|---|

## 改后稿
...
```

## 标题优化流程

1. 确认类型和核心对象。
2. 提炼读者收益或冲突。
3. 生成 8 个标题：服务型 2 个、冲突型 2 个、反问型 2 个、情绪型 2 个。
4. 标出最推荐 1 个，并说明风险。
5. 高风险事实不得写成确定标题。

## 禁止事项

- 不要把所有类型合成万能模板。
- 不要把所有小编合成平均风格。
- 不要冒充 `事儿哥` 或任何小编本人。
- 不要复制原文句子、标题或段落。
- 不要编造事实、采访、评论、官方回复、价格、时间、路线、阅读量。
- 不要把未证实爆料写成定责。
- 不要为了像账号而滥用粗口、地域攻击或扣帽子。

## 事实核验规则

- 政策/补贴：核文件名、主管单位、日期、适用对象、金额和期限。
- 医疗/健康：只做提醒，不给诊疗建议。
- 法律/维权：不定责，不公开隐私，不写未经授权当事人信息。
- 商家/消费：核价格、地址、营业状态、优惠条件和广告边界。
- 灾害/突发：只写已确认信息，未确认项明确标注。
- 图片/截图：没有授权或来源不明时只描述“有线索”，不当成事实铁证。

## 风格自检清单

- 标题是否有承德、本地对象或明确读者收益？
- 开头三段是否抓住对象、冲突和为什么要看？
- 正文是否有事实台账支撑，而不是顺滑脑补？
- 是否保留类型结构，而不是写成平均公众号稿？
- 如果使用 `事儿哥`路线，是否有强判断、人味、短段落和城市情绪？
- 是否在样本不足小编上做了提醒？
- 是否把事实和判断分开？
- 是否没有冒充、没有复制、没有编造？
