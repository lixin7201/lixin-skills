# 大宜宾公众号 Writing DNA

## 一句话 DNA

把宜宾本地的变化、求助、人物和生活线索，写成“有对象、有现场感、有普通人关系、有评论入口”的公众号稿。

## 默认写作流程

1. 先定读者承诺：消息、机会、避坑、共鸣、长脸、记忆、站队、帮忙。
2. 再定素材等级：强写长稿，中写观察，弱写短提醒/征集。
3. 选择文稿类型 DNA。
4. 如果用户指定甘甘/小美/加菲/大艳，再叠加对应小编 DNA；未指定就只用账号基线。
5. 写前建立事实台账：已确认、可推断、禁止补。
6. 输出前检查：标题对象、前三句、段落功能、事实风险、结尾动作。

## 标题基因

- “再见 + 地名/学校/老厂/街区”用于城市记忆和变化。
- “宜宾 + 对象 + 火了/来了/开放/开通”用于消息和文旅。
- “曝光/求助/急寻”只能用于授权清楚的民生线索。
- “不能没有/终于/真的要”用于观点或城市情绪，但正文必须给事实支撑。

## 开头基因

- 不从政策背景开头，从宜宾人的感受或具体对象开头。
- 可用第一人称，但只能在有真实作者/现场身份时使用。
- 常见开口：一个反常识判断、一个具体变化、一个记忆触发、一个读者问题。

## 正文结构基因

```text
对象/变化前置
→ 为什么宜宾人会在意
→ 已确认事实逐步展开
→ 场景或图片承接
→ 普通人关系
→ 风险/未知边界
→ 评论或行动入口
```

## 素材基因

- 喜欢本地地名、学校、桥、街、老厂、江边、菜市、公交、高铁、商圈。
- 喜欢用网友评论、APP 帖子、现场图、老照片做情绪支点。
- 对数字敏感，但数字必须有来源。
- 现场细节越具体，越不能编。

## 价值判断

- 城市变化值得被看见，但旧记忆也要被安放。
- 普通人的难处要被认真对待，不消费苦难。
- 宜宾人可以骄傲，但不要空喊口号。
- 公众号稿要让读者有参与感，而不是只让编辑完成表达。

## 黑名单

- 不冒充真实小编署名。
- 不把编辑署名当确定作者。
- 不搬外部来源稿做个人风格样本。
- 不补路线、停车、价格、采访、经营、政策材料、官方结论。
- 不把弱素材硬写成完整攻略、曝光长文或 98 分头条。
