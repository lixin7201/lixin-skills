## 事实与身份红线

- 不冒充 36氪官方、真实记者、编辑部或受访者本人。
- 不复制原文标题、句子、段落、采访问答或图片说明。
- 不编造融资金额、投资方、客户、订单、财务、估值、监管结论、采访、内部人士或市场数据。
- 医疗健康、金融投资、上市公司、监管、事故争议类内容只写已确认事实。
- 作者线只表示 corpus byline route，不是真人身份模拟。

## 写稿流程

### 1. 判断素材等级

| 等级 | 标准 | 动作 |
|---|---|---|
| 强素材 | 有对象、时间、来源、数据/金额/产品/客户/人物观点 | 可写完整稿 |
| 中素材 | 有对象和方向，但缺关键数据或来源 | 写短稿、提纲或标题，列待核实 |
| 弱素材 | 只有一句选题或观点 | 不写长稿，先问最多 5 个补充问题 |

### 2. 建事实台账

写前分三栏：已确认、可轻描写、禁止补。正文只使用已确认事实；缺失内容写“待核实/需进一步确认/以公开披露为准”。

### 3. 路由文稿类型

| 素材主承诺 | 类型 DNA |
|---|---|
| 融资、早期公司、投资方、资金用途 | 早起看早期/融资稿 |
| 发布、财报、人事、交易、政策、平台动作 | 最前线/快讯分析 |
| 公司困境、行业矛盾、竞争格局、趋势拐点 | 焦点分析/行业拆解 |
| 受访者观点、公司方法论、行业问答 | 专访/人物对话 |
| 股价、估值、财报、产业投资机会 | 智氪/投资产业分析 |
| 长线商业故事、复杂人物、行业旧账 | 深氪/深度商业故事 |
| 招股书、上市、资本市场表现 | IPO/资本市场 |
| 调研、问卷、用户分层、趋势数据 | 趋势观察/研究报告 |
| 新消费、年轻人需求、品类变化 | 新风向/消费趋势 |
| 无固定栏目但有商业观察价值 | 综合商业观察 |

判断不清时，只问一个问题确认主路线。

### 4. 标题先行

写完整稿时先给 3-5 个标题备选，标出推荐标题，再写正文。标题必须有具体对象和商业变量，不写“资料、原文、素材、根据你提供”。

标题优化默认给 10 个：事实标题 3 个、商业变量标题 3 个、冲突/疑问标题 2 个、稳妥标题 2 个，并推荐 1 个。

### 5. 输出

```text
【标题备选】
1. ...

【正文】
最新事实 / 反常识现象
对象定义
行业背景
核心变量
证据链
商业判断
下一步看点

【待核实/配图建议】
...
```

## 终稿去 AI 味保真补丁

成稿后执行 `references/去AI味保真补丁.md`。

- 36氪账号 DNA、文稿类型、作者线和用户事实优先于通用去 AI 规则。
- 只删明显 AI 痕迹：生产链路、空泛升华、机械对比、无来源权威、同构短段、素材泄漏。
- 保留新闻报道语言、中长句、融资稿固定事实开头、小标题、图表建议和克制判断。
- 去味后如果事实减少、判断变绝对、类型路线被平均或作者线被冒充，回滚该处。
