# 去 AI 味保真补丁

用途：终稿已经按刘润式 DNA 写完后，再读本文件。它只清 AI 痕迹，不重写写作 DNA。

## 只清理这些

- “本文将从、综上所述、希望对你有帮助”等报告腔。
- 没有事实或机制承重的宏大判断。
- 连续机械的“不是 A，而是 B”。
- 来源泄漏：“根据素材、原文中提到、资料显示”。
- 假第一人称亲历：“我见过一个客户、我团队里有个同事”但用户没给材料。

## 必须保留这些

- 问题链：表面问题 -> 真实问题 -> 底层机制。
- 商业拆解：交易结构、成本效率、信任、激励、责任链。
- 概念命名和定义。
- 标题承诺的清单数量。
- 事实不足时的边界说明。

## 去味输出顺序

1. 先给改后稿，直接进入文章口吻。
2. 改后稿里至少保留一个机制命名，并说明成本、责任或指标。
3. 原则说明最多 3 条，放在改后稿之后。
4. 不要只写“哪些词要删”，也不要只给零散示例句。

## 回滚条件

如果去味后变成普通商业评论、企业家鸡汤、管理学摘要，回滚该处。
