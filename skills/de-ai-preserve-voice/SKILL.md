# 去 AI 味保真补丁

这是终稿补丁，不是写稿引擎。先让当前写稿 skill 按自己的选题判断、思考框架、结构模板和语言 DNA 完成初稿；最后才执行本补丁。

## 上位规则

1. 当前写稿 skill 的 DNA 永远优先：标题方法、开头路径、结构骨架、句式习惯、作者思考框架、账号读者关系都不能被本补丁覆盖。
2. 用户原稿的思想优先：核心判断、论证顺序、语气强弱和可识别表达不要被改成“更通用、更正确”的模型腔。
3. 事实保真优先：数字、时间、地点、人名、机构、价格、引用、链接、指令、字段、文件名和已确认素材不为自然感牺牲。
4. 质量优先：去 AI 味不能让稿子变短、变软、变白、变成口水话；如果去味后不像原 skill，立刻回滚该处。

## 执行顺序

1. 锁定不可改项：事实、数字、来源、用户原判断、目标 skill 的风格 DNA、文章类型、小编/作者线。
2. 只找最明显的 5-10 个 AI 痕迹：空泛开场、路标句、无源权威、机械二元对比、同构段落、过度碎段、二创痕迹、顺滑总结、假亲历、假共情、官话、抽象价值词堆叠。
3. 最小补丁：能删一个词不重写一句；能改一句不重排一节；能保留作者句法就不换成自己的句法。
4. 长文默认 bounded：如果删除会改变结构、压缩超过 10% 或伤害节奏，只列“建议删除/建议改写”，不擅自大幅砍稿。
5. 二次回读：先看事实有没有变，再看是否仍像目标 skill；两个都通过，才看 AI 痕迹是否下降。

## 常见 AI 痕迹与修法

| 痕迹 | 修法 |
|---|---|
| “本文将、首先其次最后、综上所述” | 删除路标，直接进入对象、动作或判断 |
| “这不仅是 A，更是 B”连续出现 | 只保留真正推进观点的一处，其余改成事实或动作 |
| “值得注意的是、业内普遍认为”无来源 | 删除，或改成“从已给材料只能看出” |
| “资料中提到、原文中提到、根据你提供的素材、文中写到、上面说到” | 删除二创痕迹，把已确认事实直接写进作者/账号口吻；只有真实新闻来源时才写“公开信息显示、某部门通报、某平台公告、记者了解到”等来源名 |
| 一段一个抽象结论 | 补回已给素材中的具体对象、场景、动作或关系 |
| 连续很多段都只有一句短判断 | 合并同一功能的事实、解释、案例或方法段，保留真正用于停顿、梗、金句和转折的短段 |
| 结尾升华太顺 | 收回到边界、动作、余味或用户原判断 |
| 假第一人称、假朋友、假采访 | 删除；没有素材就改成“有一种常见情况” |
| 全文像同一模板换词 | 换段落功能，不换作者 DNA |

## 段落节奏保真

短段落是一种常见公众号 DNA，但“每句话都独立成段”也常是 AI 过度模仿的痕迹。终稿检查时，按目标作者的真实语料还原节奏，而不是把“短段落”理解成“全部碎段”。

- 保留作者真实的短段用途：停顿、反手判断、梗、金句、转折、分节承诺。
- 合并同一功能的连续短段：事实链、解释链、案例描述、方法步骤、来源说明，优先合并成一个自然段。
- 如果连续 5 段以上都低于 20 字，逐段判断是否真的承担不同功能；没有功能差异的，合并。
- 如果普通正文里超过一半段落都低于 20 字，把它当作过度碎段预警；这不是硬性分数线，但必须回看是否把解释链拆碎了。
- 如果一组排比、清单、例子或方法步骤只是同一层级的信息，不要一项一段；合并为一个自然段，或用真正的列表承载。
- 回归检查看三个数：段落中位长度、普通正文 <=20 字短段占比、连续 <=20 字短段最大值；生成稿应接近目标语料，不能只接近“公众号短句印象”。
- 合并后不能改变观点强弱、事实边界、作者句法和文章推进顺序。
- 歌词、标题组、清单、报价、表格、互动引导和本来就按行呈现的格式不套用本规则。

## 二创痕迹清理

成稿要像作者直接在写，而不是像 AI 在转述用户材料。

- 禁止在正文里暴露写作来源链：`资料中提到`、`原文中提到`、`根据你提供的素材`、`文中写到`、`上面说到`、`用户给的资料显示`、`参考资料里`。
- 用户给的素材只是事实台账，进入正文后要改成作者视角：直接写人、事、地点、数据、争议和判断。
- 如果确实有新闻事实或权威来源，可以保留来源属性，但必须换成读者能理解的真实来源：`公开信息显示`、`市场监管部门通报`、`平台公告称`、`记者了解到`、`商家页面显示`。
- 不确定来源真实性时，不要把“资料显示”伪装成权威来源；改成“从目前掌握的信息看”“这一点还需要进一步核实”或删掉。
- 清理后不得改变事实边界：不能为了像作者本人写，就补采访、现场、亲历、网友评论或官方回应。

## 作者 DNA 豁免

- 如果目标 skill 本来就依赖某种句式，例如刘润式“不是 A，而是 B”、洞见式数字分节、半佛式机制反转、歌词里的复沓押韵，不要一刀切删除。
- 如果目标 skill 本来就高频短段，仍要保留短段；只修正机械连续碎段，不把稿子改成传统长段议论文。
- 只处理“机械堆叠、连续复用、没有推进”的地方。
- 判断标准不是“像不像 AI 检测器喜欢”，而是“是否削弱目标 skill 的真实写法”。

## 输出契约

- 写稿任务：默认只输出最终稿，不暴露本补丁过程。
- 改稿/诊断任务：可输出“保留项、AI 痕迹、修改原则、改后稿”。
- 不确定某处会伤害文风时，标为“建议改”，不要直接替换。

## 禁止

- 不要为了去 AI 味改变立场。
- 不要新增事实、案例、数据、亲历或引用。
- 不要把强风格改成平庸口语。
- 不要把所有作者都改成同一种“自然人说话”。
- 不要为迎合检测器牺牲文章质量。
