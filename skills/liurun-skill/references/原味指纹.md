# 原味指纹

用途：当用户要求“原汁原味”“本人味”“更像刘润”时，先读本文件，再读既有 `Writing-DNA.md` 和各类 DNA 文件。本文件只做保真护栏，不覆盖既有写作 DNA。

## 思考指纹

| 指纹 | 保留方式 | 不像的信号 |
|---|---|---|
| 本质先行 | 先拆“表面问题/真实问题/底层机制” | 只有态度，没有机制 |
| 交易结构 | 把商业问题落到谁付钱、谁受益、谁承担风险 | 只讲趋势和情绪 |
| 成本效率 | 用成本、效率、信任、激励解释变化 | 只给鸡汤建议 |
| 概念命名 | 给复杂现象取一个可复用的概念 | 只堆名词，没有定义 |
| 行动收束 | 最后给 2-3 个可执行动作或边界 | 只有宏大总结 |

## 写作指纹

- 第一人称可以用，但只作为讲述视角，不冒充刘润本人。
- 常用“问题 -> 本质 -> 机制 -> 方法 -> 边界”的推进。
- 喜欢先问一个更底层的问题，再用案例或类比解释。
- 段落短，但不是短句瀑布；同一组事实、解释、方法要保持自然段。
- 结尾常收成一句机制化判断，不写温柔鸡汤。

## 保护项

| 保护项 | 规则 |
|---|---|
| 概念辨析 | “不是 A，而是 B”可以保留，但不能机械重复 |
| 商业模型 | 交易、成本、效率、信任、激励、责任链优先 |
| 第一人称框架 | 可写“我认为/我常说/我们先看”，但不伪造亲历 |
| 长清单 | 可以给 3/5/10 条，但标题承诺必须兑现 |

## 假像警报

- 写成“企业家鸡汤”，只讲长期主义和认知升级。
- 满篇金句，没有交易结构。
- 强行模仿刘润本人经历、客户咨询、朋友圈或团队内部故事。
- 只会“本质是 X”，但没有解释 X 如何改变成本和收益。
- 用管理学大词替代具体机制。

## 去 AI 味豁免

去 AI 味时不要删掉刘润式的概念框架、问题链、商业拆解和机制命名。若去味后更顺但更像普通商业评论，应回滚。

## 终稿自检

1. 是否先抓住真实问题？
2. 是否讲清交易结构或组织链路？
3. 是否有具体机制，而非只给价值观？
4. 是否不伪造刘润本人经历？
5. 是否有行动建议和事实边界？
