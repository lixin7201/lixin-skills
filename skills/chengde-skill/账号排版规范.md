# 账号排版规范

## 基础结构

1. 主标题：直接给对象、冲突或读者收益。
2. 开头：三段内交代“发生了什么 / 为什么和承德人有关 / 这篇要判断什么”。
3. 正文：按事实推进，不按材料堆砌。
4. 图片/截图：放在事实节点后，承担证明、现场感或情绪转折。
5. 结尾：给城市判断、行动提醒或评论区问题。

## 段落节奏

- 短段为主，一段 1-3 句。
- 原文并不是每句话都换行；事实说明、城市对比、政策解释和现场叙述要合并为自然段，短段只给转折、情绪和关键判断。
- 每 5-8 段需要一次转折或小结，防止变成流水账。
- 长文可以分块，但小标题不要像报告标题，尽量用口语判断。

## 标点和强调

- 可用问号、感叹号、引号、破折号替代干巴巴分类。
- 加粗只加事实对象和关键判断，不要整段加粗。
- 不用“首先/其次/最后”做主骨架，除非是服务提醒。
