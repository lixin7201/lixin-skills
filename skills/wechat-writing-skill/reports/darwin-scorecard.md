# Darwin Scorecard

评估方式：静态审查 + 12 条测试提示覆盖 + 2 条出稿样例 + holdout 维度对照。未使用独立外部评委，因此分数用于本地迭代，不等同于线上发布胜率。

| 维度 | 分数 | 判断 |
|---|---:|---|
| 事实可靠性 | 9.6 | 有素材等级、事实台账、禁止补事实清单 |
| 作者准确性 | 10.0 | 明确禁止把编辑署名当作者，四个小编均标证据等级 |
| 多小编路由 | 9.1 | 支持账号基线、类型 DNA、小编 DNA 三层路由 |
| 写稿方法可执行性 | 9.0 | 从素材判断到输出格式有完整流程 |
| Holdout 保护 | 9.4 | 原文保留，不用于批量复写训练 |
| 不像诊断能力 | 8.8 | 有七项诊断框架，后续可增加真实错稿样本 |
| 类型覆盖 | 8.9 | 七类文稿均覆盖，观点观察和人物故事仍可补样本 |
| 出稿样例质量 | 8.7 | 能按流程出稿，仍需真实素材多轮人工审稿 |
| 运行中立性 | 9.5 | 未发现特定工具或运行时绑定文本 |

综合分：9.22 / 10，折算 92.2 / 100。

## 结论

达到 Darwin 发布阈值 85 分以上，可以作为“蒸馏版大宜宾写稿 skill”进入本地调用。

## 限制

- 这不是对真实小编个人声纹的强认证。
- 这不是对所有历史稿件的人工逐篇审稿。
- 后续如果要做个人级蒸馏，应继续扩大 A 类硬证据样本。
