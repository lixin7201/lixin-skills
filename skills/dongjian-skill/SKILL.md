# 洞见 skill

基于用户本地 451 篇洞见公众号语料蒸馏。目标是按洞见常见写稿方式出稿、改稿、审稿，不冒充洞见本人或团队。

## 触发词

- 用洞见风格写
- 按洞见的写法改
- 写成洞见那种稿子
- 这篇哪里不像洞见
- 帮我按这个账号风格重写
- 用这个小编的写法写一篇
- 按我们公众号常用风格写
- 标题改成洞见风格
- 开头改得更像洞见

## 写文章标题先行规则

只要用户要求写文章、完整稿、公众号稿或按洞见风格成稿，先输出标题备选，再写正文；这条规则覆盖后文里只写【主标题】【副标题】的旧默认格式。

写正文前，必须先读标题 DNA，并站在洞见式标题风格里，从多个角度起标题：具体痛点、关系/人性判断、行动收益、强判断、数量承诺或反常识。给 3-5 个标题备选，标出“推荐标题”，再进入正文。

标题不得暴露生产链路，不写“资料、原文、素材、二创、改写、根据你提供”；标题承诺的数量、对象和行动必须在正文兑现。

## 工作流

### Step 1：判断任务类型

新稿写作、原稿改写、标题优化、开头优化、审稿反馈、风格统一、短内容扩写、长文压缩、选题判断、多稿对比。

### Step 2：检查材料是否足够

缺少以下硬信息时，先列缺口，不直接编造：时间、地点、人物、价格、政策、活动信息、数据来源、采访来源、商家信息、权威说法。

STOP：医疗、法律、金融、未成年人、暴力、隐私、公共政策、商家权益等事实敏感题，缺权威材料时只写生活提醒或补采清单。

### Step 3：选择文章骨架

先判断素材真正承载的痛点，再选骨架。骨架只决定推进方向，不固定标题句式、开头顺序或每节句型。

- 人生观点/关系/金钱：从具体失序处推到认知、边界、习惯或心气。
- 新闻/影视/作文/人物：只取最能照见普通人处境的一刀。
- 职场：从具体处境推到留痕、确认、反馈、边界和证据，不把所有职场稿都写成三法则。
- 方法/清单：清单服务行动，不把每条写成同一种判断句；标题承诺了多少条，就必须交付多少条。
- 家庭/教育/亲密关系：从关系变化推到边界修复，不只写“付出/委屈/清醒”；要写出“过度承担 -> 被默认 -> 委屈账本 -> 控制/怨气 -> 关系反噬 -> 重新分工”的链条。
- 敏感新闻/暴力/法律风险：只借已确认事实做入口，不补写未核实前史；主判断落到边界、法律意识和家庭秩序。

### Step 4：正式出稿

同时做到：标题强判断，开头快速入题，正文数字分节，材料服务观点，语言短句推进，转折从故事到人性，结尾回扣标题并给行动或姿态；但故事、案例和解释链不要被切成一串微段。

不要为了“像洞见”强行套同一副骨架。跨题材写作时，先复刻洞见的判断方式，再让句式、节奏和段落形状跟着素材变化。

默认输出公开正文：

```text
【主标题】
【副标题】
【正文】
```

正式正文里不要输出“开头”“正文”“结尾”“第一部分”等元标签；这些只是写作阶段的提示，不是给读者看的文案。标题后直接进入正文，结尾用自然收束或 `▽` 前收束。

### Step 5：自检后再输出

- 是否只是普通 AI 文？
- 是否只模仿口吻，没有模仿结构？
- 是否缺少洞见常用的故事/判断/行动链条？
- 是否用了洞见不会用的官话、学术话、模板话？
- 是否为了像洞见而重复同一套标题公式、开头路径或判断链条？
- 标题承诺的数量、对象和行动是否兑现了？
- 敏感事件是否避免补写未核实细节，只保留公共判断？
- 职场稿是否有具体的留痕、确认、反馈、边界动作，而不是只写情绪？
- 是否编造事实？
- 是否有冒充本人或团队风险？
- 如果用户要求原汁原味，是否通过 `references/原味指纹.md` 的处境、判断、行动链自检？

### Step 6：终稿去 AI 味保真补丁

最后执行 `de-ai-preserve-voice`，只清理模板腔，不删除洞见式写法。

- 保留数字分节、故事/场景、清醒判断、普通人处境、行动建议和结尾姿态。
- 只处理“本文将”“更有温度更有逻辑”、无对象金句、连续同句式收束和没有材料支撑的升华。
- 不删除洞见式常用句式；只在同类句式连续复用、不能推进关系时降密度。
- 洞见原文短段多，但故事段和解释段会自然展开；长文回归目标：段落中位约 20-30 字，<=20 字短段不宜明显高过原文三成多，连续 <=20 字短段超过 5 段时合并同功能故事链、判断链和行动链。
- 成稿必须去掉二创痕迹：不要写“资料中提到、原文中提到、根据你提供的素材、文中写到”；把事实改成自然叙述，再落到普通人的处境、边界和行动。
- 不把故事改成讲义，不把行动改成鸡汤，不为了去味减少标题承诺的条数。
- 去味后如果不像洞见，或少了故事-判断-行动链，回滚该处。

## 写稿规则

### 标题

标题必须先来自素材的具体痛点，再参考标题骨架。常见骨架包括：`一个人最大的 X：Y`、`人在单位：A、B、C`、`为什么 X？答案在 Y`、`普通人变强的开始：Y`、`看了 X，才知道 Y`、`X 不是 A，而是 B`。

同一批标题不要连续套同一种骨架；不同题材不能只替换关键词。每个标题都要说出该素材独有的痛点、关系或动作。

### 开头

前 3-8 段完成：短判断 -> 故事/场景 -> 核心困惑 -> 本文主判断。不要写“本文将分析”。

### 正文

用 `01/02/03` 数字分节。每节：案例进入 -> 抽象判断 -> 读者处境 -> 行动/警句。

数字分节可以稳定，但每节的小标题和推进动作要跟素材长出来。不要让每篇都变成“01 原因 / 02 影响 / 03 怎么办”，也不要把“开头/结尾”当可见段落名。

抽象判断前尽量有一个可感知的生活细节或动作：饭桌、聊天框、工位、房间、转账、药盒、作息、衣服、钥匙、会议记录。细节用于落地，不用于编造硬事实。

遇到标题承诺清单数量时，正文必须兑现数量。可以分组呈现，也可以每条短写，但总数不能少；不要把“50 条”改写成 20 条散文。

遇到家庭关系稿，正文要写关系如何一步步变坏：不是“付出多所以委屈”一句话，而是从承担、默认、期待落空、怨气累积、关系反噬，最后落到边界和分工。

家庭过度付出题要写“双边误读”：付出者以为自己是在牺牲和撑家，家人感受到的可能是冷脸、讨债、控制、摆功劳；家人以为这是理所当然，付出者感受到的是被忽视和被亏欠。洞见式判断要从这个错位里长出来。

遇到职场稿，至少有一节落到可执行自保动作：文字确认、节点反馈、责任边界、证据留存、延期说明、利益分配。不要只写“清醒、体面、别内耗”。

遇到敏感新闻稿，不复盘未知细节，不推测人物动机，不编造第一次/多年积怨/邻居说法。用“公开材料显示”“这类事件提醒我们”保持距离，再转入边界、法律和普通家庭的止损。

敏感新闻里也不要用“第一次忍了、第二次忍了、第三次忍了”这类修辞链条，因为它会暗示未确认的重复前史。可以改写成“如果家庭长期用忍让替代规则，风险就会被一步步放大”。

### 常用句式

不是 A，而是 B；越是 A，越要 B；表面上是 A，本质上是 B；一个人的 X，藏着他的 Y；到了一定年纪才懂，A 比 B 更重要。

这些句式是语气校准，不是填空题。连续两段出现同类句式时，必须换成具体场景、动作或关系变化来推进。

### 反模板化自检

- 这篇文章的主判断是否只能用于当前素材，而不是换个词就能套到任何题？
- 标题、开头、每节收束是否都在复用同一种句型？
- 每一节是否有不同的推进动作：场景、反差、关系、代价、行动，而不是同一套“案例 -> 判断 -> 行动”机械重复？
- 如果把题目换成职场/家庭/金钱，文章骨架是否仍一模一样？如果是，重写骨架。
- 稳定的是洞见的世界观：把普通人的痛翻译成清醒判断；变化的是材料入口、段落形状和句子落点。

### 禁用句式

随着社会的发展、本文将从、综上所述、希望本文对你有所帮助、更有温度更有逻辑、让我们共同努力。

### 素材

用熟悉故事打开情绪，用概念增加解释力，用普通人场景落地。每节 1-2 个材料，不堆材料。

### 转折

从现象推到深层原因，从个体故事推到普通人处境，从情绪共鸣推到清醒行动。

### 结尾

在 `▽` 前收束：回扣标题，给 2-4 句短判断，落到边界、体面、心气、行动或推荐。不要编造阅读数、二维码、互动数据。

## 改稿流程

1. 保留原稿已确认事实。
2. 找出原稿主判断，把它压成洞见式标题。
3. 删除泛泛铺垫，改成故事/场景开头。
4. 重排为 3-5 个数字分节。
5. 每节补足“案例 -> 判断 -> 普通人动作”。
6. 删除 AI 腔、官话、空泛金句。
7. 输出改后版本。

## 审稿诊断模块

用户给稿子时，必须输出：

- 哪些地方像。
- 哪些地方不像。
- 标题差在哪里。
- 开头差在哪里。
- 结构差在哪里。
- 语言差在哪里。
- 素材使用差在哪里。
- 结尾差在哪里。
- 具体怎么改。
- 改后版本。

- 不要用“更有温度、更有逻辑、更有故事感”替代具体修改。
- 不要把通用 AI 文包装成洞见风格。
